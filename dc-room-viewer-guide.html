DC Room Viewer — How-To Guide
Data-centre room layout & visualisation
Open the tool →

DC Room Viewer is a to-scale layout editor for data-centre rooms and halls. Place racks, PDUs, cooling units, sensors and cameras from a component library onto a raised-floor tile grid, draw walls, doors, cages and containment, colour equipment by any metadata field, watch live capacity totals in the Summary panel, and export the drawing as PNG or PDF. Everything is stored in plain JSON files on your machine — nothing leaves your browser.

1. Rooms and libraries

Two JSON file formats, one simple relationship.

A component library (dc-room-library) defines reusable equipment types — racks, PDUs, cooling units, structural items — each with a physical footprint in millimetres, a colour, and a metadata schema (rack ID, customer, capacity, and so on). The tool ships with a built-in starter library, and you can open your own via Library ▾ → Open library…. Racks carry standard contractedKw / avgKw / peakKw power fields and a usageType (leased, compensation, unoccupied, planned, reserved, unusable, internal, unbuilt); the Monitoring & safety category holds round point markers — CCTV cameras, temperature/humidity/smoke/leak sensors, door contacts, PIR motion, access readers and exit signs.

A room (dc-room) is the drawing itself: the hall dimensions, the tile grid, and every placed object. When you place a component, its definition is copied into the room file, so a room always opens standalone — you never need the original library to view or edit it. Library ▾ → Export room components as library turns a room's embedded definitions back into a shareable library file.

All coordinates and dimensions are in millimetres. The default grid is the standard 600 × 600 mm raised-floor tile; the pitch is configurable per room from the Layers panel.

2. Getting started

  1. New creates a room — give it a name and its width/depth in metres.
  2. Click a component in the palette (left sidebar), then click the canvas to place it. The mode stays armed so you can click out a whole row; R rotates before placing, Esc exits.
  3. Zoom with the mouse wheel (zooms to the pointer), pan with Space+drag or middle-drag, press 0 to fit the room.
  4. Save As… writes the room to a .json file. In Chrome and Edge the file stays linked, so Save overwrites it in place.

3. Placing and editing equipment

Equipment snaps to the tile grid by whichever edge the grid meets — a 600 mm rack lands tile-aligned at any rotation. Hold Alt while dragging or placing to bypass snapping.

  • Select with a click; Shift-click to add/remove; drag on empty floor for a rubber-band selection.
  • Move by dragging, or nudge one tile with the arrow keys (Shift+arrows = 10 mm).
  • Rotate in 90° steps with R, the rotation handle, or the context menu.
  • The Inspector shows exact x/y coordinates, rotation, the display label, and all metadata fields — rack ID, customer, contracted/average/peak kW, plus any custom field you add.
  • The white tick on a rack marks its front face.

The properties dialog

Double-click any object — rack, wall, door, cage, zone, column, label, dimension or sensor — to open its properties dialog: the object's data fields plus a metadata editor (every object type can carry metadata, not just equipment). Edits apply as a single undo step when you press Save; Esc discards them. Enter with one object selected, or Properties… in the context menu, opens the same dialog.

Right-click any object for the context menu: properties, rotate, duplicate, lock, bring to front/send to back, cage membership, delete.

4. Walls, doors, cages, zones and dimensions

Walls

The Wall tool draws a polyline: click each vertex, double-click or Enter to finish. Select a wall to edit its vertices — drag the handles, Alt+double-click a segment to insert a vertex, Alt-click a handle to remove one. (Plain double-click opens the wall's properties.) Thickness is set in the Inspector. The room's outer boundary wall is drawn automatically from the room shape.

Doors

The Door tool places a door on the nearest wall, aligned to it, with a dashed swing arc. Hinge side, swing direction and swing angle are editable in the Inspector. Doors re-seat onto the nearest wall when dropped near one.

Cages, containment and restricted areas

Cage, Cold aisle, Hot aisle and Restricted draw closed polygons the same way as walls. Cages have auto-capture: equipment dropped inside becomes a member (shown in the Inspector), and dragged out it leaves. Each cage or zone can be given its own boundary colour from the Inspector or properties dialog. Columns are placed with a single click.

Dimension annotations

The Dimension tool (M) draws a double-headed measuring arrow with two clicks; the length label ("14.40 m") updates automatically as you drag either endpoint handle. Dimensions live on the Labels & legends layer and export with the drawing.

5. Room shape and hall coordinates

Non-rectangular rooms

Rooms are rectangular by default, but real halls have annexes and notches. Edit room outline (in Room properties, shown in the Inspector when nothing is selected, or under the Grid layer's controls) turns the boundary into an editable polygon: drag the corner handles, double-click an edge to add a corner, Alt-click one to remove it, then Enter to finish (Esc cancels). The tile grid, boundary wall and floor-area figure all follow the outline.

Grid coordinate labels

Column letters run across the top of the hall and row numbers down the side, exactly as on survey drawings; the status bar shows the tile reference under the cursor (e.g. AF12). Toggle them, or skip confusable letters (many halls omit I and U), from the Grid layer's controls.

Room properties

With nothing selected, the Inspector edits the room itself: name, site, updated-by, contracted power (kW), usable floor area, floor-loading note and free-text notes. These feed the Summary panel and the export title block.

A fixed, standard stack — objects always live on the layer their type belongs to.

7. Groups and cages

Select several objects and press Ctrl/Cmd+G to group them — clicking any member then selects the whole group (double-click drills into a single member); Ctrl/Cmd+Shift+G ungroups. Groups are the right tool for rack rows you want to move as one.

Cage membership is spatial parent-child: it follows from an object's position inside the cage polygon (with auto-capture on), and is stored on each object as cageId. Deleting a cage releases its members.

8. Status overlays and the key

The Capacity / status overlay section tints equipment by any metadata field:

  • Categories — one colour per distinct value (e.g. customer). Colours are assigned automatically and can be overridden per value.
  • Numeric thresholds — banded colours for fields like contractedKw (e.g. green ≤ 4, amber ≤ 8, red above).

The Colour by usage type button applies the conventional survey palette in one click — pink/red leased, purple compensation, green unoccupied, yellow reserved, grey unusable, light-blue internal, and so on — driven by the racks' standard usageType field.

The key is drawn on the canvas itself (drag it anywhere; its position is saved in the room file), so it appears in PNG/PDF exports, and each entry shows a live count — "Leased (96)". With no overlay active it shows the equipment category colours instead.

9. The Summary panel

The Summary sidebar section aggregates the room live as you edit: floor area (outline-aware) and usable area, rack count with a usage-type breakdown, total contracted / average / peak kW across racks — with a utilisation percentage against the room's contracted power when set — PDU and cooling capacity totals, a per-customer table (racks and contracted kW), and the monitoring device count. A condensed version ("266 racks · Σ contracted 312 kW of 330 kW (95%)") is printed in the export title block.

10. Saving, autosave and files

  • In Chrome/Edge, Open and Save As… use real file handles — Save then overwrites the file in place. In other browsers, Save downloads the file instead.
  • Unsaved work is snapshotted to the browser's local storage every second while you edit. After a crash or accidental tab close, reopening the tool offers to restore the snapshot.
  • Opening the same file in two tabs shows a warning — edits are not shared between tabs, and saving in one overwrites the other's changes.
  • Opening a library file via Open offers to load it into the palette instead.

11. Exporting PNG and PDF

Export ▾ → PNG image renders the room at print quality with a title block — room name, site, dimensions, floor area, contracted power, grid pitch, export date, updated-by, a capacity summary line and a scale bar — plus the on-canvas key. Export ▾ → PDF drawing wraps the same rendering on an A3 page (orientation chosen automatically), with the room name as the document title.

Exports always include everything visible on unhidden layers, regardless of the current zoom. Hide layers first to produce, say, an equipment-only drawing.

13. JSON file reference

Both formats are versioned; unknown fields survive a load/save round-trip.

Component library (dc-room-library)

{
  "format": "dc-room-library",
  "version": 1,
  "name": "Standard DC components",
  "components": [
    {
      "id": "rack-600x1200",
      "name": "Rack 600×1200",
      "category": "rack",   // rack | pdu | cooling | structural | monitoring | other
      "footprint": { "w": 600, "d": 1200 },   // mm
      "visual": { "fill": "#5B7A9D", "stroke": "#3A4A5D",
                  "shortCode": "R", "front": "south" },
      "resizable": false,
      "metadata": { "uSpace": 42 },    // defaults copied to new instances
      "metadataSchema": [
        { "key": "rackId",       "label": "Rack ID",         "type": "string" },
        { "key": "customer",     "label": "Customer",        "type": "string" },
        { "key": "usageType",    "label": "Usage type",      "type": "select",
          "options": ["leased", "compensation", "unoccupied", "planned",
                      "reserved", "unusable", "internal", "unbuilt"] },
        { "key": "contractedKw", "label": "Contracted (kW)", "type": "number" },
        { "key": "avgKw",        "label": "Average (kW)",    "type": "number" },
        { "key": "peakKw",       "label": "Peak (kW)",       "type": "number" }
      ]
    }
  ]
}
category: "monitoring" components render as round point markers on the Monitoring layer; schema fields with "type": "select" render as a dropdown of their options.

14. Privacy

All data handling happens locally in your browser. Room and library files live on your machine; the crash-recovery snapshot lives in your browser's local storage. The page's Content-Security-Policy blocks network access except to the pinned CDN libraries and fonts — your layout data cannot leave the browser.
