DC Room Viewer — How-To Guide
Data-centre room layout & visualisation
Open the tool →

DC Room Viewer is a to-scale layout editor for data-centre rooms and halls. Place racks, PDUs and cooling units from a component library onto a raised-floor tile grid, draw walls, doors, cages and containment, colour equipment by any metadata field, and export the drawing as PNG or PDF. Everything is stored in plain JSON files on your machine — nothing leaves your browser.

1. Rooms and libraries

Two JSON file formats, one simple relationship.

A component library (dc-room-library) defines reusable equipment types — racks, PDUs, cooling units, structural items — each with a physical footprint in millimetres, a colour, and a metadata schema (rack ID, customer, capacity, and so on). The tool ships with a built-in starter library, and you can open your own via Library ▾ → Open library….

A room (dc-room) is the drawing itself: the hall dimensions, the tile grid, and every placed object. When you place a component, its definition is copied into the room file, so a room always opens standalone — you never need the original library to view or edit it. Library ▾ → Export room components as library turns a room's embedded definitions back into a shareable library file.

All coordinates and dimensions are in millimetres. The default grid is the standard 600 × 600 mm raised-floor tile; the pitch is configurable per room from the Layers panel.

2. Getting started

  1. New creates a room — give it a name and its width/depth in metres.
  2. Click a component in the palette (left sidebar), then click the canvas to place it. The mode stays armed so you can click out a whole row; R rotates before placing, Esc exits.
  3. Zoom with the mouse wheel (zooms to the pointer), pan with Space+drag or middle-drag, press 0 to fit the room.
  4. Save As… writes the room to a .json file. In Chrome and Edge the file stays linked, so Save overwrites it in place.

3. Placing and editing equipment

Equipment snaps to the tile grid by whichever edge the grid meets — a 600 mm rack lands tile-aligned at any rotation. Hold Alt while dragging or placing to bypass snapping.

  • Select with a click; Shift-click to add/remove; drag on empty floor for a rubber-band selection.
  • Move by dragging, or nudge one tile with the arrow keys (Shift+arrows = 10 mm).
  • Rotate in 90° steps with R, the rotation handle, or the context menu.
  • The Inspector shows exact x/y coordinates, rotation, the display label, and all metadata fields — rack ID, customer, capacity, plus any custom field you add.
  • The white tick on a rack marks its front face.

Right-click any object for the context menu: rotate, duplicate, lock, bring to front/send to back, cage membership, delete.

4. Walls, doors, cages and zones

Walls

The Wall tool draws a polyline: click each vertex, double-click or Enter to finish. Select a wall to edit its vertices — drag the handles, double-click a segment to insert a vertex, Alt-click a handle to remove one. Thickness is set in the Inspector. (The room's outer boundary wall is drawn automatically from the room dimensions.)

Doors

The Door tool places a door on the nearest wall, aligned to it, with a dashed swing arc. Hinge side, swing direction and swing angle are editable in the Inspector. Doors re-seat onto the nearest wall when dropped near one.

Cages, containment and restricted areas

Cage, Cold aisle and Restricted draw closed polygons the same way as walls. Cages have auto-capture: equipment dropped inside becomes a member (shown in the Inspector), and dragged out it leaves. Columns are placed with a single click.

5. Layers

A fixed, standard stack — objects always live on the layer their type belongs to.

LayerContentsControls
GridTile grid and room boundaryShow/hide, pitch, origin, snap on/off

ContainmentCages, cold-aisle containment, restricted areasShow/hide, lock, per-cage visibility
EquipmentRacks, PDUs, coolingShow/hide, lock

Labels & legendsFree text labels and the on-canvas keyShow/hide, lock

Locking a layer makes everything on it unselectable and immovable. Individual objects can also be locked from the Inspector or context menu — locked objects refuse deletion until unlocked.

6. Groups and cages

Select several objects and press Ctrl/Cmd+G to group them — clicking any member then selects the whole group (double-click drills into a single member); Ctrl/Cmd+Shift+G ungroups. Groups are the right tool for rack rows you want to move as one.

Cage membership is spatial parent-child: it follows from an object's position inside the cage polygon (with auto-capture on), and is stored on each object as cageId. Deleting a cage releases its members.

7. Status overlays and the key

The Capacity / status overlay section tints equipment by any metadata field:

  • Categories — one colour per distinct value (e.g. customer). Colours are assigned automatically and can be overridden per value.
  • Numeric thresholds — banded colours for fields like capacityKw (e.g. green ≤ 4, amber ≤ 8, red above).

The key is drawn on the canvas itself (drag it anywhere; its position is saved in the room file), so it appears in PNG/PDF exports. With no overlay active it shows the equipment category colours instead.

8. Saving, autosave and files

  • In Chrome/Edge, Open and Save As… use real file handles — Save then overwrites the file in place. In other browsers, Save downloads the file instead.
  • Unsaved work is snapshotted to the browser's local storage every second while you edit. After a crash or accidental tab close, reopening the tool offers to restore the snapshot.
  • Opening the same file in two tabs shows a warning — edits are not shared between tabs, and saving in one overwrites the other's changes.
  • Opening a library file via Open offers to load it into the palette instead.

9. Exporting PNG and PDF

Export ▾ → PNG image renders the room at print quality with a title block — room name, dimensions, grid pitch, export date and a scale bar — plus the on-canvas key. Export ▾ → PDF drawing wraps the same rendering on an A3 page (orientation chosen automatically), with the room name as the document title.

Exports always include everything visible on unhidden layers, regardless of the current zoom. Hide layers first to produce, say, an equipment-only drawing.

11. JSON file reference

Both formats are versioned; unknown fields survive a load/save round-trip.

Component library (dc-room-library)

{
  "format": "dc-room-library",
  "version": 1,
  "name": "Standard DC components",
  "components": [
    {
      "id": "rack-600x1200",
      "name": "Rack 600×1200",
      "category": "rack",              // rack | pdu | cooling | structural | other
      "footprint": { "w": 600, "d": 1200 },   // mm
      "visual": { "fill": "#5B7A9D", "stroke": "#3A4A5D",
                  "shortCode": "R", "front": "south" },
      "resizable": false,
      "metadata": { "uSpace": 42 },    // defaults copied to new instances
      "metadataSchema": [
        { "key": "rackId",     "label": "Rack ID",       "type": "string" },
        { "key": "customer",   "label": "Customer",      "type": "string" },
        { "key": "capacityKw", "label": "Capacity (kW)", "type": "number" }
      ]
    }
  ]
}

Room (dc-room)

{
  "format": "dc-room",
  "version": 1,
  "meta": { "name": "Hall 1", "site": "", "notes": "",
            "created": "…", "modified": "…" },
  "room": {
    "width": 24000, "depth": 18000,          // mm
    "grid": { "pitch": 600, "originX": 0, "originY": 0,
              "show": true, "snap": true },
    "wallThickness": 200
  },
  "defs": { "rack-600x1200": { …component copied from library… } },
  "objects": [
    { "id": "o-a1b2", "type": "equipment", "defId": "rack-600x1200",
      "x": 1500, "y": 1200,                  // footprint centre, mm
      "rotation": 0, "label": "A01",
      "groupId": "g-row-a", "cageId": "c-acme",
      "metadata": { "rackId": "A01", "customer": "Acme", "capacityKw": 8 } },
    { "id": "o-w1", "type": "wall",
      "points": [[6000,0],[6000,9000]], "thickness": 150 },
    { "id": "o-d1", "type": "door", "x": 3000, "y": 0, "width": 900,
      "rotation": 0, "hinge": "left", "swing": "in", "swingAngle": 90 },
    { "id": "o-c1", "type": "column", "x": 6000, "y": 6000,
      "w": 400, "d": 400, "shape": "rect" },
    { "id": "c-acme", "type": "cage", "name": "Acme cage",
      "points": [[1200,600],[7200,600],[7200,6600],[1200,6600]],
      "autoCapture": true },
    { "id": "z-1", "type": "zone", "kind": "cold-aisle",   // or "restricted"
      "name": "Cold aisle 1", "points": [ … ] },
    { "id": "l-1", "type": "label", "text": "ROW A",
      "x": 4000, "y": 900, "fontSize": 300, "rotation": 0 }
  ],
  "groups": [ { "id": "g-row-a", "name": "Row A" } ],
  "layers": { "building":    { "visible": true, "locked": false },
              "containment": { "visible": true, "locked": false },
              "equipment":   { "visible": true, "locked": false },
              "status":      { "visible": true },
              "labels":      { "visible": true, "locked": false } },
  "statusOverlay": {
    "enabled": true, "mode": "numeric", "field": "capacityKw",
    "opacity": 0.55,
    "categorical": { "overrides": { "Acme": "#4A5D47" } },
    "numeric": { "stops": [
      { "upTo": 4,    "color": "#43a047", "label": "≤ 4" },
      { "upTo": 8,    "color": "#f9a825", "label": "≤ 8" },
      { "upTo": null, "color": "#e74c3c", "label": "above" } ] }
  },
  "legend": { "visible": true, "x": 18800, "y": 600 }
}
Coordinates use millimetres with the origin at the room's top-left corner, x increasing rightwards and y downwards. Equipment x, y is the footprint centre; rotation is degrees clockwise about that centre.

12. Privacy

All data handling happens locally in your browser. Room and library files live on your machine; the crash-recovery snapshot lives in your browser's local storage. The page's Content-Security-Policy blocks network access except to the pinned CDN libraries and fonts — your layout data cannot leave the browser.
