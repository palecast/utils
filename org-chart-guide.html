1. Get your org in

Import an Excel or CSV export from your HR system — column names are matched flexibly.
  1. Click Open in the header and pick a .xlsx, .xls or .csv file. Only the first sheet of a workbook is read.
  2. Check the warning banner: duplicate Role IDs and circular reporting lines are flagged immediately.
  3. Switch to the Chart tab to see the hierarchy.

Column headers are matched case-insensitively against a set of aliases, so most HR exports work without renaming anything. The full alias list is in the Reference section — for example Manager, Reports To and Manager ID all map to the same field.

  • Role ID is the only structural requirement — every row needs one, and the Manager column must contain the manager's Role ID. Rows with a blank Role ID are auto-assigned one (R1, R2…) on import.
  • A role whose Manager is blank (or unknown) becomes a top-level root. Multiple roots are grouped under a synthetic "Organization" node.
  • Status is normalised to one of: filled, vacant, pending start, planned, on hold, sunsetting, removed, earmarked. Anything unrecognised becomes planned.
Just exploring? Click Load test data on the Table tab to populate the app with a realistic sample org, including tags and dotted-line managers.

2. Work with files like documents

Open, Save, Save As — plus a crash-recovery net underneath.

The header works like a desktop app: the centre shows the open file name, with a leading • whenever in-memory data has diverged from the saved file. In Chromium browsers (Chrome, Edge), Save writes back to the original file in place; other browsers download a fresh copy instead.

  • Save / Save As… always write every role and every column — they are your master copy, unaffected by filters.
  • Unsaved changes… shows exactly what differs from the last save — the same added / removed / moved / changed breakdown as Compare.

Crash recovery

While you have unsaved work, a snapshot is continuously kept in the browser's local storage (per tab). If the tab crashes or is closed before saving, the next visit shows a banner offering to Restore the work. The snapshot is cleared whenever you save or open a file, and old snapshots are removed after 7 days.

Same file in two tabs

Tabs announce which file they have open. If two tabs hold a file with the same name, both show an amber warning — edits are not shared between tabs, and saving in one overwrites the other. Close one tab, or make sure you know which is current.

Autosave is a crash-recovery net, not cloud sync. Your file on disk only changes when you press Save.

3. Edit the org

Edit in the grid, in a form, or directly on the chart — everything is undoable.

On the Table tab

  • Click any cell to edit it in place. Header filters under each column name narrow the grid as you type.
  • Changing a role's Manager opens a tree picker mirroring the hierarchy — the role's own subtree is excluded, so a reporting cycle cannot be created.
  • The ✎ row button (or double-clicking a row) opens the role editor dialog, which edits every field at once and saves as a single undo step.
  • Row actions: +child adds a direct report, ⎘ duplicates the role, ✕ deletes it (its reports move up to its manager).
  • Keyboard: arrow keys move the active cell, Enter opens the editor, and typing a letter in a list cell jumps to the first matching option.

On the Chart tab

  • Right-click any card for: Edit details, Add child, Add sibling, Rename, View subtree chart, Add/Remove dotted manager, Delete.
  • Drag a card onto another to change its manager. Drops that would create a cycle are refused; moving a manager with their whole team asks for confirmation first.
  • Double-click a card to open the role editor.

Undo / Redo

Cmd/Ctrl+Z undoes the last structural change; Cmd/Ctrl+Shift+Z redoes it. Both are also buttons in the sidebar's History section. The last 20 steps are kept.

Sketch live in a meeting: drag teams around on the chart while people watch, then undo back to reality — or Save As to keep the scenario.

4. Navigate a large chart

Find anyone in two keystrokes; keep hundreds of roles readable.
  • Zoom: sidebar − / + buttons, the mouse wheel, or hotkeys — + and - zoom, 0 resets to 100%, 1 fits the whole visible chart to the canvas.
  • Depth: the sidebar Depth control limits how many levels are expanded — step down to an executive summary, back up to full detail.
  • Collapse badges: a collapsed branch shows a badge with its hidden subtree count. Click the badge to open a popup mini-chart of that subtree — drill deeper via its own badges, use the breadcrumb to step back, and double-click a card to locate it in the main chart.
  • Find: the search box on the Chart tab matches Role ID, job title, incumbent name and department. Press Enter (or ▲/▼) to cycle through matches — collapsed ancestors expand automatically and the chart pans to each hit.
  • Show from: focus the whole app (chart, pivot, summary, exports) on one subtree. Pick a role from the sidebar tree picker or click a node, then follow the breadcrumb trail back up.

5. Slice, filter and colour

Cut a big org down to the exact slice you need — the three mechanisms compose.
  • Filters: two field filters in the sidebar (Status, Department, Location, Gender, Employment Type, Working Pattern) are ANDed together — e.g. Department = Engineering AND Status = vacant.
  • Tags: a free-form tags field on every role (space- or comma-separated, shown as chips on cards). The Tags filter matches any selected tag, and combines with the field filters.
  • Colour: colour cards by any category field. The legend is draggable — park it anywhere; it snaps back when you change the colour dimension. Colours never repeat, however many values the field has.
  • Visible Fields: choose which fields appear on cards, including computed roll-ups — subtree headcount, cumulative cost, and per-status counts.
Planning scenario: tag roles 2027-plan as you work, then filter to that tag to review just the proposed changes — combined with "Show from" to stay inside one division.

6. Matrix (dotted-line) reporting

Secondary reporting lines without cluttering the tree.

Any role can carry a list of dotted-line managers alongside its solid line. They appear three ways:

  • As dashed pills on the role's card — clicking a pill pans to that manager;
  • As a column on the Table tab (Role IDs separated by ;, which round-trips through CSV/Excel);
  • As a dashed overlay drawn between the linked cards — enable Show dotted lines in the sidebar.

Add or remove dotted lines from the chart's right-click menu via a searchable role picker. Imports are validated: self-references, unknown IDs, duplicates, the role's own primary manager and anything that would create a dotted cycle are scrubbed, with a notification explaining what was dropped and why.

7. Run an org review with Compare

Quantify exactly what changed between two versions of the org — e.g. this month's HR extract vs last month's.
  1. Open your baseline file (last month's org, or your working plan).
  2. Click Compare… in the header and pick the second file (e.g. the latest HR extract).
  3. Read the summary chips: headcount and total cost before → after, with deltas.
  4. Work through the four sections — Added (in your current data but not the file), Removed (in the file but not your data), Moved (manager changed), and Changed (field-by-field edits).
  5. Click any row for a role that still exists to jump straight to its chart node.
  6. Click Export diff CSV for an auditable change list to circulate for sign-off.

Compare is read-only — it never modifies your data. Roles are matched by Role ID, so both files need consistent IDs.

Also useful before a reorg lands: compare your planned-state file against production to verify it contains only the intended moves.

8. Delegate a subtree and Merge it back

Send a department head just their team, let them edit, then apply their changes — without them ever seeing salary data.
  1. Focus the chart on the department with Show from (or filter to it).
  2. On the Chart sidebar, choose 📊 Data… → Excel or CSV. The field picker opens — Cost to Company is unticked by default, so salaries stay behind unless you explicitly include them. Role ID is always included so the file merges back cleanly.
  3. Send the file. The recipient edits it in Excel — retitling roles, moving people, adding rows.
  4. When it comes back, click Merge… in the header and pick their file.
  5. Review the plan: chips show what will be added, updated, what is already in sync, and any skipped rows (no Role ID). Each item lists the exact field changes.
  6. Apply items one by one with their Apply buttons, or Apply all remaining. Every apply is a single undo step.
  7. Optionally Export plan CSV to file the merge decision trail.

Merge is deliberately conservative:

  • It never deletes — roles missing from the subset file are left untouched (use Compare if you need to see removals).
  • It is column-aware — only fields whose column exists in the file are compared or written. A file exported without the Cost column cannot zero out salaries.

9. Share and export

What you see is what you share — exports follow the current filters and focus.

All exports live in the Chart sidebar's Export section (Data exports are also on the Table tab). The rule of thumb: Save writes everything; exports write what you're looking at.

ExportWhat you getBest for
🖨 PrintBrowser print of the visible chart with the colour key as a footer, colours forced onPaper handouts, PDF via the print dialog
🎨 Image… → PNGHigh-resolution snapshot including the colour key stripEmail, chat, wikis
🎨 Image… → SVGVector version of the chart — crisp at any zoomSlide decks, large-format print
📊 Data… → CSV / ExcelThe visible rows (filters, tags and focus applied) with a column picker — cost excluded unless tickedSending a slice to someone; re-importable subsets for Merge
📋 PowerPoint SmartArtAn indented text outline of the visible hierarchy on your clipboardNative, editable PowerPoint org charts

Filtered exports stay coherent

If a role's manager is filtered out of view, the exported file rewrites that role's manager to its nearest visible ancestor — so the exported hierarchy always hangs together and re-imports cleanly.

Pasting into PowerPoint

  1. Click 📋 PowerPoint SmartArt — the outline is copied to your clipboard.
  2. In PowerPoint: Insert → SmartArt → Hierarchy → Organization Chart.
  3. Open the SmartArt Text Pane, select its placeholder lines, and paste.

10. Analyse with Pivot

Cross-tab headcount or cost by any two dimensions.

The Pivot tab builds a cross-tab from the same slice the chart shows (focus, filters and tags all apply). Choose the Rows and Columns dimensions, and aggregate either headcount (count) or cost (sum). Export CSV/Excel writes the rendered cross-tab — exactly the table you see — ready to paste into a monthly report.

Example: Rows = Department, Columns = Status, Aggregate = Count → a vacancy heat-map by department in three clicks.

11. Where your data lives

Everything stays on your machine.
  • All processing happens in your browser. Org data is never uploaded — the only network requests are for the page's fonts and libraries.
  • Your master copy is the file you Save. Autosave snapshots and UI preferences live in the browser's local storage on your machine, and snapshots are garbage-collected after 7 days.
  • Clearing browser site data removes snapshots and preferences — never your saved files.
  • Salary data leaves the tool only when you explicitly tick Cost in an export, or use Save/Save As (which always writes all fields).
